Actúa como un Senior EHR UI/UX Architect. Tu objetivo es refactorizar documents.html para convertirlo en un gestor de archivos clínico optimizado para pantallas de 14'', utilizando componentes de PrimeNG 15 y el sistema modular existente.

1. Arquitectura y Shell Modular
Integración: Utiliza layout_list.js para inyectar automáticamente el Header, el Navigation Sidebar y el Patient Banner.

Tipografía: Aplica estrictamente el CSS existente: textos generales a 1.3rem (--fs-main) y el título de la página en el Header a 2rem (--fs-title).

Título: El título central en el Header debe ser "Patient Documents".

2. Sistema de Filtros (Tab View)
Crea una barra de navegación de pestañas (puedes usar p-tabMenu o una estructura de botones de alta densidad) inmediatamente debajo del banner del paciente con las siguientes opciones:

Filtros: "All", "Reports", "Informed Consent", "Various", "Digital History", "Signed Documents".

Estilo: Texto en azul, con una línea inferior resaltada para la pestaña activa, manteniendo la fuente en 1.3rem.

3. Barra de Acciones y Búsqueda
A la derecha de la barra de filtros o en una línea inferior, añade:

Buscador: Un componente p-inputText con icono pi-search que ocupe el 30% del ancho, con el placeholder "Search documents...".

Contador: Un badge azul que muestre el número total de documentos encontrados.

4. La Rejilla de Documentos (Componente p-table)
Implementa un componente PrimeNG p-table con las siguientes columnas recuperadas de la lógica clínica:

Name: El nombre del documento como un link azul (ej. TTO_INP_CARD_Dr.Rory Rogers).

Author: Nombre del profesional que generó el archivo.

Department: Servicio de origen (ej. Cardiology, Adult Emergency).

Type: Categoría (ej. TTO Report, Medical Report, Discharge Report).

Date: Fecha y hora de creación.

Status: Usa p-tag con colores semánticos:

Signed: Verde (success).

Draft: Naranja (warning).

Access Web: Icono de mundo (pi-globe) si el documento es accesible externamente.

Actions: Grupo de iconos compactos para Download (pi-file-pdf), Delete (pi-trash) y Share (pi-share-alt).

5. Optimización de Espacio
Configura la tabla para alta densidad: padding mínimo en las celdas para visualizar al menos 15 documentos sin hacer scroll en una laptop de 14''.

Asegura que el contenedor de la tabla tenga scroll vertical y la cabecera sea fija (sticky).
